# Phase 5 Plan

> 阶段名称：番茄小说提取 Agent
> 目标：验证“搜索 + 用户确认 + 内容抽取 + RAG 入库”型工作流
> 对应设计：[phase-5-novel-ingest.md](/Users/litiezhu/workspace/github/youmeng-gateway/docs/architecture/phase-5-novel-ingest.md)

## 一、阶段目标

本阶段是第一个偏内容处理型 agent。重点不是回答问题，而是把外部内容可靠地拉入内部知识库。

完成后应具备：

1. 根据小说名检索候选结果
2. 支持用户确认目标小说
3. 抽取正文或章节数据
4. 将内容切分并写入 RAG

## 二、范围

### 本阶段要做

- 小说搜索接口封装
- 候选结果展示
- 用户确认流程
- 内容抽取与分片
- RAG 入库

### 本阶段不做

- 多站点聚合抓取
- 大规模并发采集
- 高级去重和去噪

## 三、核心任务

### Task 5.1 搜索与候选结果

- 输入小说名
- 返回候选列表

### Task 5.2 用户确认

- 确认选中小说
- 记录确认结果

### Task 5.3 内容抽取

- 提取章节或正文
- 记录抽取状态

### Task 5.4 RAG 入库

- 文本切分
- 元数据生成
- 写入向量库

## 四、阶段产出

- Novel Ingest workflow
- 用户确认机制
- RAG 入库链路

## 五、阶段通过标准

- [ ] 可搜索并返回候选小说
- [ ] 用户确认后可抽取内容
- [ ] 内容可成功写入 RAG
