- `Feishu` 作为主入口，`Web/API` 作为诊断、运维和备用入口
- 按配置和请求选择任意单个有权限的 GitHub 仓库
- 单用户单任务优先，明确队列化或忙碌语义
- 保持唯一主链 `Feishu/API -> main-agent -> ralph -> code-review-agent -> delivery`
- 保持 `LLM + MCP + skill first`，工程代码只保留队列、状态真相、超时、诊断和 delivery gate
