# Archive

本目录只存放已经失效、已完成或仅用于历史追溯的文档。

当前归档分为三类：

- `architecture/`: 历史或过渡期架构方案
- `plans/`: 已完成或已失效的阶段性执行计划
- `status/`: 仅用于保留旧阶段的状态快照，不作为当前 continuity file

归档文档保留的目的只有两个：

- 解释历史决策是怎么来的
- 便于回溯某一轮收口或迁移做过什么

归档文档不承担以下职责：

- 不作为当前架构入口
- 不作为当前演进方向
- 不作为当前状态真相
- 不作为下一轮实现的默认设计依据

如果某份归档文档里的结论仍然有效，应先提炼后搬到公开主文档树，而不是继续直接引用归档原文。

## 归档规则

一份文档应进入 `archive/`，当且仅当满足以下任一条件：

- 已被新的 canonical 文档完整覆盖
- 只描述某一轮阶段性执行拆分
- 只记录当时的设计推演，而不是当前仍成立的边界
- 继续放在主文档树会误导后续 agent 把历史材料当成当前真相

归档前应先做两件事：

1. 在主文档树补出新的 current source of truth
2. 在 `docs/README.md` 中更新阅读入口和历史映射

归档后应满足：

- 任何 agent 不需要读取 archive 也能继续实现当前目标
- archive 只承担历史解释，不承担当前指挥
