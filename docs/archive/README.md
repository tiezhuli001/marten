# Archive

本目录只存放已经失效、已完成或仅用于历史追溯的文档。

当前归档只保留两类少量材料：

- `architecture/`: 仍有解释价值、且当前 canonical 文档未完整替代的少量历史架构方案
- `codebase-audits/`: 某一轮仓库收口或减法决策的审计快照

归档文档保留的目的只有两个：

- 解释历史决策是怎么来的
- 便于回溯某一轮收口或迁移为何这样取舍

归档文档不承担以下职责：

- 不作为当前架构入口
- 不作为当前演进方向
- 不作为当前状态真相
- 不作为下一轮实现的默认设计依据

如果某份归档文档里的结论仍然有效，应先提炼后搬到公开主文档树，而不是继续直接引用归档原文。纯执行流水、阶段性 rollout checklist、已完成后不再解释当前结构的历史计划，默认直接删除，不长期存档。

## 归档规则

一份文档应进入 `archive/`，当且仅当满足以下任一条件：

- 已被新的 canonical 文档完整覆盖
- 只描述某一轮阶段性执行拆分
- 只记录当时的设计推演，而不是当前仍成立的边界
- 继续放在主文档树会误导后续 agent 把历史材料当成当前真相

如果某份历史计划只剩阶段性执行痕迹、对当前继续实现没有解释价值，直接删除，不强求长期保留。当前仓库已按此规则删除 `2026-03-22/23` 的历史 rollout 计划。

归档前应先做两件事：

1. 在主文档树补出新的 current source of truth
2. 在 `docs/README.md` 中更新阅读入口和历史映射

归档后应满足：

- 任何 agent 不需要读取 archive 也能继续实现当前目标
- archive 只承担历史解释，不承担当前指挥
- archive 不应因为“也许以后想看”而继续膨胀
